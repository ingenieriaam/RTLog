RTLog

hardware description language for synchronous and asynchronous circuits

### Agenda: --------------- - RTLog Language. - The RTLog compiler. - RTLog applied to asynchronous circuits.

What is this language?

 

- It's a Hardware Description Language.

- It's simpler than Verilog or VHDL in the sense syntax doesn't have overhead clauses.

- Is pure RTL, doesn't consider behavorial constructions.

- It can describe some kind of async circuits easily.

We work with the 4 phases protocol as handshake between latches. ![async](img/async.svg) vvvv ### An additional step in the compiler -------------------- - Our compiler first generates a graph representing a sync circuit. - In order to build an async circuit the previous graph is modified. hhhh ### Transforming sync graph to async -------------------- We can summarize this task in these steps - Find all flip-flop in the circuits. - Replace the flip-flops by latches. - For each latch we add a linear ( latch ) controller. - Find all register-to-register paths. - Connect request and acknowledges signals each other. - Add input and output handshake ports. - Connect this ports to the corresponding latches. hhhh ### Multiple paths converge to a latch ---------------------------- A request to the latch only should be issued only when all paths do. To achieve this behaviour we use a join controller.
